# 模块一：用户账户与认证模块 
    核心功能：处理所有与用户身份相关的事务。

    包含内容：

    后端：用户注册/登录API、JWT令牌生成与验证、密码加密（BCrypt）、个人信息管理。

    前端：登录页、注册页、个人中心页面（修改资料、管理地址）。

    数据库：用户表、收货地址表。

    特点：基础且关键，是其他模块的依赖。

# 模块二：菜单与商品管理模块
    核心功能：管理餐厅的核心产品——菜品。

    包含内容：

    后端：菜品的增、删、改、查API；菜品分类管理；设置菜品状态（上架/下架）。

    前端：

    用户端：菜单展示页面（按分类、搜索）、菜品详情。

    管理端：菜单管理页面（可操作所有菜品）。

    数据库：菜品分类表、菜品详情表。

    特点：数据驱动，以CRUD操作为主。

# 模块三：购物车与订单核心流程模块
    核心功能：处理从“加入购物车”到“生成订单”的核心业务流程。

    包含内容：

    后端：购物车商品增、删、改、查API；生成订单API（整合购物车、用户、地址信息）；订单状态流转逻辑（待处理 -> 制作中...）。

    前端：购物车页面、订单确认页面、订单状态概览。

    数据库：购物车项表、订单主表、订单明细表（关联菜品）。

    特点：业务逻辑最复杂的模块之一，涉及多个数据实体的关联和状态管理。

# 模块四：餐厅运营与后台管理模块
    核心功能：为餐厅员工提供管理后台。

    包含内容：

    后端：

    订单管理API：获取订单列表、更新订单状态（如接单、出餐）。

    营业管理API：开关接单状态。

    促销活动API：创建折扣、套餐（依赖模块二）。

    员工账户管理API（管理员用）。

    前端：管理后台的登录页、订单管理面板、营业开关、促销活动设置页面、员工管理页面。

    数据库：促销活动表、员工/管理员账户表。

    特点：后台系统，注重操作效率和状态实时性。

# 模块五：支付模拟与客户反馈模块
    核心功能：处理交易的闭环与用户评价。

    包含内容：

    后端：

    支付模拟API：模拟支付成功/失败回调，触发订单状态更新。

    评价与反馈API：提交评价、获取菜品/餐厅评分。

    数据统计API（基础）：如销量统计（依赖模块三）。

# 前端：

用户端：模拟支付页面、订单评价页面。

管理端：客户反馈看板、销售数据图表页面。

数据库：支付记录表、评价表。

特点：两个相对独立但重要的子功能，共同构成用户体验的终点。

# 模块零：系统公共模块
核心功能：所有模块依赖的公共部分。

包含内容：

后端：全局配置（餐厅名称、LOGO）、API响应格式统一、异常处理、工具类（如日期、加密）。

前端：导航栏、页脚、全局样式、网络请求封装。

数据库：数据库初始设计、连接配置。

项目：Git仓库管理、基础项目搭建、最终集成与部署。
